# GenAi MERN Interview Planner

A full-stack MERN app that:
- registers and logs in users
- accepts a resume upload plus job description
- generates an interview prep report with Gemini
- stores reports in MongoDB
- lets the user download a tailored resume PDF

## Stack
- Frontend: React + Vite
- Backend: Node.js + Express + MongoDB + Mongoose
- AI: Google Gemini via `@google/genai`

## Folder structure
- `frontend/` - React client
- `backend/` - Express API

## Backend setup
```bash
cd backend
cp .env.example .env
npm install
npm run dev
```

### Required backend environment variables
```env
PORT=5000
MONGO_URI=mongodb://127.0.0.1:27017/genai-mern
CORS_ORIGIN=http://localhost:5173
ACCESS_TOKEN_SECRET=replace_with_a_long_random_secret
ACCESS_TOKEN_EXPIRY=1d
REFRESH_TOKEN_SECRET=replace_with_another_long_random_secret
REFRESH_TOKEN_EXPIRY=7d
GEMINI_API_KEY=replace_with_your_gemini_api_key
GENAI_MODEL=gemini-2.5-flash
NODE_ENV=development
```

## Frontend setup
```bash
cd frontend
cp .env.example .env
npm install
npm run dev
```

### Frontend environment variables
```env
VITE_API_BASE_URL=http://localhost:5000
```

## Common run flow
1. Start MongoDB locally or use MongoDB Atlas.
2. Start the backend on port `5000`.
3. Start the frontend on port `5173`.
4. Open the frontend in the browser.
5. Register a user, log in, then generate an interview report.

## Notes
- Resume uploads support PDF and DOCX.
- Downloaded resume PDFs are generated by Gemini + Puppeteer.
- Keep cookies enabled because authentication uses HTTP-only cookies.


## Windows install note
If `npm install` fails because of registry or lockfile issues, make sure npm is using the public registry:

```bash
npm config set registry https://registry.npmjs.org/
```

Then delete `node_modules` and any `package-lock.json` files before reinstalling.
